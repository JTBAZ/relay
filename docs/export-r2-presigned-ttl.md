# Export / R2 presigned URL time limits (P8-sec-004)

## What is actually “signed”?

- **Patron library export HTTP** — `GET /api/v1/export/media/:creator_id/:media_id/content` (and thumb/preview) is served **by Relay**, not as a direct time-limited R2 URL. Access uses Relay’s own checks (and optional tier gate). There is **no** separate “signed export GET TTL” in that path.
- **Creator browser uploads** — the client receives a **presigned `PUT`** URL from `POST /api/v1/relay/upload/init` (see ADR 002). That URL is generated by the AWS SigV4 presigner (`presignR2Put` in `src/storage/relay-upload-r2.ts`).

## Presigned PUT TTL

| Setting | Meaning |
|--------|---------|
| `R2_PRESIGN_EXPIRES_SEC` | Lifetime of the presigned **PUT** URL in seconds. **Clamped** to **60–14,400** (1 minute to 4 hours). If unset or invalid, Relay uses **900** (15 minutes). |

Code: `getPresignExpiresSec()` in `src/storage/relay-upload-r2.ts`.

## After expiry (replay)

When the signing window ends, **Cloudflare R2** (S3-compatible) treats the request as unauthorized. Clients typically see **HTTP 403** with an AccessDenied-style error, not a success. **Reusing the same URL after it has expired** should fail; trying again before expiry may still succeed (same object, idempotent PUT).

For automated checks without calling R2, Relay exposes `presignedUrlSigningExpiresAt` / `isPresignedUrlExpired` parsing `X-Amz-Date` and `X-Amz-Expires` on the URL — the same parameters R2 uses to enforce the window.
