# Webpage-level design list (visual scope only)

Feature-level rows stay in **UI Planning — Inventory**. Screen-level rows for v0 / design handoff live in **UI Planning — Design Pages** (`tbliRw7EDiZBOLL2z`).

## How to use

- Sort **Design Pages** by **Roadmap Rank** for design sprints.
- From **Inventory**, use **Primary design page** to see which screen skin a feature belongs to (many inventory rows → one page).

## What was excluded on purpose

Micro-interactions that are not “a page” (e.g. sync pill, bulk bar, triage as a dialog) roll up under the **Creator library**, **Patreon & data sync**, or **Action center** **visual** system—they remain in Inventory for engineering.
